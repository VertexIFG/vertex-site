# Vertex Infrastructure Group Site

Single-page marketing site for `vertexifg.com` — the "AS-BUILT" redesign. The page reads like a
precision engineering document for an underground route: an image-free typographic hero with a
perpetually drilling bore line, a scroll-driven plan-view WebGL flythrough, a pinned 2D as-built
bore profile drawing, and an equipment spec ledger.

## Stack

- Vite + React 19 + TypeScript
- GSAP (ScrollTrigger, SplitText) + Lenis smooth scroll
- React Three Fiber / three.js — lazy-loaded scene chunk, never in the initial payload
- Hand-written CSS design tokens (`src/styles/tokens.css`), self-hosted woff2 fonts
- Playwright regression suite (10 spec files, both desktop + mobile projects)

## Local Development

```bash
npm ci
npm run dev
```

Site runs at `http://127.0.0.1:5173`.

## QA

```bash
npm run lint            # eslint
npm run build           # tsc -b + vite build (typecheck gate)
npm run test:e2e        # Playwright against the dev server
PW_PREVIEW=1 npm run test:e2e   # the release gate: full suite against the production build
```

Visual baselines live in `tests/visual.spec.ts-snapshots/` (captured on macOS chromium under
reduced motion). After an intentional visual change:

```bash
PW_PREVIEW=1 npx playwright test tests/visual.spec.ts --update-snapshots
```

Verified release state: Lighthouse desktop 100/100/100/100, mobile 93/100/100/100, CLS 0,
zero frames >32ms through both pinned chapters.

## Images & Fonts

Source images are parked in `assets-src/` (not shipped to the page). Optimized derivatives in
`public/assets/` are generated by:

```bash
npm run optimize:images
```

The script enforces per-file size budgets and builds favicons, the og:image, and responsive
webp sets. Fonts are latin-subset woff2 in `public/fonts/` (Archivo variable, Inter variable,
IBM Plex Mono 400/500).

## Motion Architecture

- `src/lib/gsapSetup.ts` registers plugins once; `src/lib/smoothScroll.ts` owns the Lenis
  singleton (StrictMode-safe).
- `src/components/MotionRoot.tsx` owns page-wide choreography (reveals, scrubs, progress bar).
- Pinned chapters: `BoreChapter` (2D canvas profile, scrubbed) and `RouteChapter`
  (R3F plan view, scrubbed; falls back to a static SVG on mobile, reduced motion,
  or missing WebGL).
- Everything degrades: `prefers-reduced-motion` disables Lenis, pins, and reveals while keeping
  all content visible (enforced by `tests/motion.spec.ts`).

## Going Live

The production build is fully static — `npm run build` emits `dist/`, deployable to any
static host. Recommended: Cloudflare Pages, since the domain's DNS already lives on
Cloudflare (nameservers `ines/tom.ns.cloudflare.com`).

**Cloudflare Pages recipe (run in the account that owns the vertexifg.com zone):**

1. Create a Pages project (e.g. `vertex-site`) and deploy:
   `npm ci && npm run build && npx wrangler pages deploy dist --project-name vertex-site --branch main`
2. In the Pages project, add custom domains `vertexifg.com` and `www.vertexifg.com`.
   Because the zone is in the same account, Cloudflare creates the DNS records and
   certificates automatically.
3. Optional CI: `.github/workflows/deploy-cloudflare-pages.yml` is ready — it deploys on
   every push to `main` once `CLOUDFLARE_API_TOKEN` (Pages:Edit) and `CLOUDFLARE_ACCOUNT_ID`
   secrets are set on the repo.

**Temporary preview** (not attached to the domain): https://vertex-site-6ki.pages.dev

**Form delivery reminder:** the first inquiry submitted on the live site emails a one-time
activation link to `inquiries@vertexifg.com` — the client clicks it once and delivery is on.

## Packaging a Client Deliverable

```bash
./scripts/package-zip.sh
```

Produces a clean source zip (git archive — no node_modules, dist, or test artifacts) in
`~/Downloads`.

## Form Delivery

The contact form posts natively to FormSubmit (`https://formsubmit.co/inquiries@vertexifg.com`)
— no backend needed, and the top-level POST passes FormSubmit's Cloudflare protection (their
AJAX endpoint does not, which is why the form intentionally avoids it). After processing,
FormSubmit redirects back to `?sent=1#contact`, which renders the inline confirmation.

**One-time setup:** the first submission emails an activation link to
`inquiries@vertexifg.com`; click it once and all subsequent inquiries arrive normally.
